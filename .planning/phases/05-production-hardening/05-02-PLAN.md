<objective>
Create the Starter.Caching and Starter.Responses class library modules, plus the ApiResponse<T> envelope type in Starter.Shared.

Purpose: Caching provides IMemoryCache and IDistributedCache infrastructure with configurable defaults. Responses provides an opt-in action filter that wraps successful responses in a standardized ApiResponse<T> envelope. The envelope type lives in Shared to avoid controller dependencies on the Responses module.

Output: ApiResponse<T> in Shared, two new class library projects with extension methods/filters/options, solution file entries. A cache-aside sample endpoint will be added in Plan 04. Program.cs wiring happens in Plan 04.
</objective>

   - `internal sealed class ApiResponseFilter : IResultFilter` (internal -- registered via DI, applied via ServiceFilter)
   - `OnResultExecuting(ResultExecutingContext context)`:
     - Guard: skip if result is not `ObjectResult`
     - Guard: skip if value is already `ApiResponse<object>` (avoid double-wrapping)
     - Guard: skip if status code is NOT 2xx (`objectResult.StatusCode is not (>= 200 and < 300)`)
     - Guard: skip if objectResult.Value is null and status is 204 No Content

   - `OnResultExecuted(ResultExecutedContext context)` -- empty body
   - XML doc: `/// Wraps successful (2xx) ObjectResult responses in an ApiResponse envelope. Applied per-controller/action via [ServiceFilter(typeof(ApiResponseFilter))]. Error responses (4xx/5xx) are NOT wrapped -- they use ProblemDetails from GlobalExceptionHandler.`
