   - Properties:
     - `public bool EnableForHttps { get; set; } = false;` with XML doc: `/// Security: Keep false to prevent CRIME/BREACH side-channel attacks. Enable only if you understand the risks and use anti-forgery tokens.`
     - `public string BrotliLevel { get; set; } = "Fastest";` with XML doc comment noting valid values: Optimal, Fastest, NoCompression, SmallestSize
     - `public string GzipLevel { get; set; } = "Fastest";`

3. Create `src/Starter.Compression/CompressionExtensions.cs`:
   - `namespace Starter.Compression;`
   - `public static class CompressionExtensions`
   - `AddAppCompression(this IServiceCollection services, IConfiguration configuration)`:
     - Register IOptions with `BindConfiguration(CompressionModuleOptions.SectionName)`, `ValidateDataAnnotations()`, `ValidateOnStart()`
     - Read options: `configuration.GetSection(CompressionModuleOptions.SectionName).Get<CompressionModuleOptions>() ?? new()`
     - Parse compression levels: `Enum.TryParse<CompressionLevel>(options.BrotliLevel, out var brotliLevel)` with fallback to `CompressionLevel.Fastest`
     - Parse compression levels: `Enum.TryParse<CompressionLevel>(options.GzipLevel, out var gzipLevel)` with fallback to `CompressionLevel.Fastest`
     - Call `services.AddResponseCompression(opts => { opts.EnableForHttps = options.EnableForHttps; opts.Providers.Add<BrotliCompressionProvider>(); opts.Providers.Add<GzipCompressionProvider>(); });`
     - Call `services.Configure<BrotliCompressionProviderOptions>(o => o.Level = brotliLevel);`
     - Call `services.Configure<GzipCompressionProviderOptions>(o => o.Level = gzipLevel);`
     - Return `services`
   - `UseAppCompression(this WebApplication app)`:
     - XML doc: `/// Adds response compression middleware. Must be called BEFORE any middleware that writes response bodies. WARNING: Enabling compression over HTTPS exposes the app to CRIME/BREACH side-channel attacks. See CompressionModuleOptions.EnableForHttps.`
     - `app.UseResponseCompression();`
     - Return `app`
