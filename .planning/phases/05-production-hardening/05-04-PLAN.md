1. Create `src/Starter.WebApi/Controllers/CacheDemoController.cs`:
   - `namespace Starter.WebApi.Controllers;`
   - `[ApiController]`, `[Route("api/[controller]")]`, `[ApiVersionNeutral]` (infrastructure demo, not a versioned business API)
   - Primary constructor: `public class CacheDemoController(IMemoryCache cache, IOptions<CachingOptions> options) : ControllerBase`
   - GET endpoint `[HttpGet("time")]`:
     - Cache key: `"cache-demo:server-time"`
     - Use cache-aside pattern:
       ```csharp
       if (cache.TryGetValue(cacheKey, out string? cachedTime) && cachedTime is not null)
       {
           return Ok(new { Time = cachedTime, Source = "cache" });
       }
       var currentTime = DateTime.UtcNow.ToString("O");
       var cacheEntryOptions = new MemoryCacheEntryOptions()
           .SetAbsoluteExpiration(TimeSpan.FromSeconds(options.Value.DefaultExpirationSeconds))
           .SetSlidingExpiration(TimeSpan.FromSeconds(options.Value.SlidingExpirationSeconds));
       cache.Set(cacheKey, currentTime, cacheEntryOptions);
       return Ok(new { Time = currentTime, Source = "generated" });
       ```
   - DELETE endpoint `[HttpDelete("time")]`:
     - Evicts the cached entry: `cache.Remove("cache-demo:server-time");`
     - Returns `NoContent()`
   - XML doc on class: `/// Demonstrates the cache-aside pattern using IMemoryCache. Hit GET /api/cachedemo/time twice -- the second call returns the cached value. Use DELETE to evict.`
   - Required usings:
     - `using Microsoft.AspNetCore.Mvc;`
     - `using Microsoft.Extensions.Caching.Memory;`
     - `using Microsoft.Extensions.Options;`
     - `using Starter.Caching.Options;`

2. Modify `src/Starter.WebApi/Controllers/TodoController.cs`:
   - Add `[EnableRateLimiting("fixed")]` attribute on the controller class (demonstrates per-endpoint rate limiting with the "fixed" policy)
   - Add `[ServiceFilter(typeof(Starter.Responses.Filters.ApiResponseFilter))]` attribute on the controller class (demonstrates response envelope wrapping)
   - Add required usings:
     - `using System.Threading.RateLimiting;` (if needed for EnableRateLimiting)
     - `using Microsoft.AspNetCore.RateLimiting;` (for EnableRateLimiting attribute)
   - Note: The ApiResponseFilter is internal in Starter.Responses, so the ServiceFilter typeof reference needs the full namespace path. Since the Host project references Starter.Responses, and the filter is registered in DI via AddAppResponses(), ServiceFilter resolves it at runtime. However, since ApiResponseFilter is internal, we cannot use typeof() from the Host project.

   CORRECTION: Because ApiResponseFilter is internal to Starter.Responses, the Host project cannot reference its type directly. Two solutions:
   (a) Make ApiResponseFilter public (breaks internal-by-default convention)
   (b) Create a public marker attribute in Starter.Responses that inherits ServiceFilterAttribute

   Use approach (b): Create a `[WrapResponse]` attribute in Starter.Responses:
   - File: `src/Starter.Responses/Attributes/WrapResponseAttribute.cs`
   - Content:
     ```csharp
     namespace Starter.Responses.Attributes;

     /// <summary>
     /// Apply to a controller or action to wrap successful responses in an ApiResponse envelope.
     /// The underlying ApiResponseFilter is resolved from DI -- requires AddAppResponses() in Program.cs.
     /// </summary>
     [AttributeUsage(AttributeTargets.Class | AttributeTargets.Method)]
     public sealed class WrapResponseAttribute : ServiceFilterAttribute
     {
         public WrapResponseAttribute() : base(typeof(Filters.ApiResponseFilter)) { }
     }
     ```
   - This is public, references the internal filter type from within the same assembly (valid), and gives controllers a clean `[WrapResponse]` attribute.

   Then on TodoController, use: `[WrapResponse]` (from `using Starter.Responses.Attributes;`)

   Also add to CacheDemoController: `[EnableRateLimiting("sliding")]` to demonstrate a different rate limiting policy.
