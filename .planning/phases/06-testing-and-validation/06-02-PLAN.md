    - Constructor receives `AuthWebApplicationFactory factory`, stores `_client = factory.CreateClient()`
    - These tests exercise the real Identity + JWT auth pipeline against the in-memory SQLite database

    Tests:
    a. `Register_WithValidCredentials_ReturnsCreatedWithToken()` -- POST `/api/auth/register` with:
       ```json
       { "email": "newuser@test.com", "password": "Test1234!", "confirmPassword": "Test1234!" }
       ```
       Assert HttpStatusCode.Created (201). Deserialize response, assert `accessToken` property exists and is not null/empty, assert `userId` property exists, assert `email` property equals `"newuser@test.com"`.

    b. `Register_DuplicateEmail_ReturnsConflict()` -- Register a user with email `"duplicate@test.com"`. Then register again with same email. Assert second call returns HttpStatusCode.Conflict (409).

    c. `Login_WithValidCredentials_ReturnsOkWithToken()` -- First register a user with email `"logintest@test.com"` and password `"Test1234!"`. Then POST `/api/auth/login` with `{ "email": "logintest@test.com", "password": "Test1234!" }`. Assert HttpStatusCode.OK (200). Assert response contains `accessToken` that is not null/empty.

    d. `Login_WithInvalidPassword_ReturnsUnauthorized()` -- Register with `"wrongpw@test.com"` / `"Test1234!"`. Then login with `"wrongpw@test.com"` / `"WrongPassword1!"`. Assert HttpStatusCode.Unauthorized (401).

    e. `Register_Login_AccessProtected_FullRoundTrip()` -- The critical end-to-end test:
       1. Register user: POST `/api/auth/register` with `{ "email": "roundtrip@test.com", "password": "Test1234!", "confirmPassword": "Test1234!" }`. Extract `accessToken` from response.
       2. Set Authorization header: `_client.DefaultRequestHeaders.Authorization = new AuthenticationHeaderValue("Bearer", token)`
       3. Access protected endpoint: GET `/api/v1/todos`. Assert HttpStatusCode.OK (200). This proves the JWT token issued by the auth pipeline is accepted by the JWT validation middleware.

    f. `AccessProtected_WithoutToken_ReturnsUnauthorized()` -- Create a NEW client from the factory (no auth headers). GET `/api/v1/todos`. Assert HttpStatusCode.Unauthorized (401).

    IMPORTANT: Each test should use unique email addresses to avoid conflicts between tests running in the same factory instance. Since AuthWebApplicationFactory uses a single SQLite in-memory DB per factory instance (shared across all tests in the class fixture), user registrations persist across tests within the same class.

    IMPORTANT: Use `System.Net.Http.Json.HttpClientJsonExtensions` for `PostAsJsonAsync` and `ReadFromJsonAsync` (available from `System.Net.Http.Json` namespace).

    Run: `dotnet test tests/Starter.WebApi.Tests.Integration --filter "AuthFlowTests" --verbosity normal`
